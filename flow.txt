CONTINUATION OF PROGRAM FLOW
│
├── Ask user if they want to run in preview mode (Dry Run)
│     ├── If YES:
│     │     ├── Do not move files
│     │     ├── Only display what actions would be taken
│     │     └── Continue to summary report
│     └── If NO:
│           └── Proceed with actual file movement
│
├── Ask user for confirmation before organizing
│     ├── If user enters NO:
│     │     └── Cancel operation and exit safely
│     └── If user enters YES:
│           └── Continue processing
│
├── During file processing:
│     ├── Ignore hidden files
│     ├── Ignore unsupported or restricted file types
│     ├── Prevent overwriting files with same name
│     │     └── Rename file if conflict occurs
│
├── Logging process:
│     ├── For every file action:
│     │     ├── Record original file name
│     │     ├── Record destination folder
│     │     └── Record timestamp
│     └── Save details in log file (log.txt)
│
├── Error handling:
│     ├── If file cannot be moved:
│     │     ├── Show warning message
│     │     └── Continue with next file
│     ├── If permission error occurs:
│     │     └── Skip file safely
│
├── Undo support (optional future feature):
│     ├── Store original file locations
│     ├── Allow user to restore files to original state
│
├── Final summary display:
│     ├── Total files scanned
│     ├── Total files moved
│     ├── Files skipped
│     ├── Errors encountered
│
├── Ask user:
│     ├── "Do you want to organize another folder?"
│     │     ├── YES → Restart program flow
│     │     └── NO → Exit program
│
└── END PROGRAM
